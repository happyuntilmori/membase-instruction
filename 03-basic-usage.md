## 기본 개념: 도구(Tools)와 자원(Resources)

멤베이스는 MCP(Model Context Protocol)를 통해 AI 에이전트와 대화해요. AI 에이전트가 멤베이스를 사용하려면 **도구(Tools)**를 호출해요.

여러분이 AI 에이전트에게 자연어로 말하면, 에이전트가 알아서 적절한 멤베이스 도구를 호출해요. 직접 도구를 호출할 필요는 없어요!

### 전체 도구 목록

**메모리 관련:**
- `search_memory` — 기억 검색
- `add_memory` (또는 `membase_store`) — 기억 저장
- `membase_forget` — 기억 삭제
- `membase_profile` — 내 프로필 보기

**위키 관련:**
- `search_wiki` — 위키 검색
- `add_wiki` — 위키 문서 생성
- `update_wiki` — 위키 문서 수정
- `delete_wiki` — 위키 문서 삭제

**유틸리티:**
- `get_current_date` — 현재 날짜/시간

**자원(Resources):**
- `membase://profile` — 내 설정과 선호도
- `membase://recent` — 최근 기억 10개

## 기억 저장하기 (add_memory)

AI 에이전트에게 이렇게 말하면 기억을 저장해요:

### 직접 저장 요청

```
"타입스크립트를 자바스크립트보다 더 선호한다는 걸 기억해줘"
"내가 함수형 프로그래밍 스타일을 좋아한다는 걸 저장해줘"
"우리 프로젝트 데이터베이스는 PostgreSQL 15야, 기억해줘"
```

### 자동 저장 (autoCapture 켜져 있을 때)

autoCapture가 활성화되어 있으면 AI가 알아서 중요한 내용을 저장해요:

```
자동 저장 트리거:
  ✅ "나는 코드에 주석을 달지 않는 스타일이야"
  ✅ "이 프로젝트 배포는 AWS EC2로 해"
  ✅ "팀장은 박민준이야"
  ✅ "저번에 그 버그는 await를 빠트려서 났어"
  
  ❌ "안녕"
  ❌ "고마워"
  ❌ "응" (50글자 미만은 저장 안 됨)
```

### 저장 타이밍

autoCapture가 켜져 있을 때 저장 시점:
- **5분 동안 대화가 없을 때** — 자동으로 이전 대화 저장
- **메시지 20개가 쌓였을 때** — 자동으로 저장
- 수동 요청 시 즉시 저장

> **크론/스케줄러 필요 없음!** 멤베이스는 완전 자동이에요. 별도의 크론 작업이나 스케줄러를 설정할 필요가 전혀 없어요.

### 좋은 저장 형식 vs 나쁜 저장 형식

```
✅ 좋은 형식 (명확하고 구체적):
"사용자는 Python 3.11을 사용하며, pytest를 테스트 도구로 선호함"
"프로젝트 A는 React + FastAPI 스택, 배포는 Vercel + Railway"
"버그 수정: useEffect 내 async 함수는 반드시 try-catch로 감싸야 함"

❌ 나쁜 형식 (모호하고 불완전):
"pytest 씀"
"프로젝트 스택"
"버그 있었음"
```

### 민감한 정보 보호

비밀번호, API 키 등 민감한 정보는 저장하지 않도록 태그로 감싸요:

```
<private>
DATABASE_PASSWORD=supersecret123
AWS_SECRET_KEY=abc123xyz
</private>
```

또는:

```
<membase-private>
이 내용은 멤베이스에 저장하지 마세요
</membase-private>
```

## 기억 검색하기 (search_memory)

저장된 기억을 꺼내는 방법이에요.

### AI 에이전트에게 물어보기

```
"내가 Python 관련해서 뭘 기억하고 있어?"
"프로젝트 A에 대해 아는 것 다 알려줘"
"내 코딩 스타일 기억나?"
"지난번에 해결한 버그들 보여줘"
"내 최근 기억 보여줘"
```

### 날짜 필터로 검색

```
"지난 주에 저장한 기억들 보여줘"
"2026년 1월에 기억한 것들 찾아줘"
```

### 자동 불러오기 (autoRecall)

`autoRecall: true` 설정 시, AI가 모든 응답 전에 관련 기억을 자동으로 검색하고 활용해요.

```
예시: autoRecall이 켜진 상태에서

사용자: "파이썬 코드 리뷰해줘"
AI 내부 동작:
  1. search_memory("파이썬 코드")
  2. 검색 결과: "사용자는 함수형 스타일 선호, pytest 사용, 타입 힌트 필수"
  3. 이 컨텍스트를 활용해서 맞춤형 코드 리뷰 제공
```

### 직접 검색 (membase://recent)

```
"membase://recent 보여줘"
```

최근 저장된 기억 10개를 시간순으로 보여줘요.

## 위키 문서 만들기 (add_wiki)

프로젝트 문서나 참조 자료를 위키에 저장하는 방법이에요.

### 기본 위키 저장

```
"다음 내용을 위키에 저장해줘: [마크다운 문서 내용]"
"이 API 문서를 위키 'project-alpha' 컬렉션에 저장해줘"
```

### 위키링크 사용

위키 문서 안에서 다른 문서를 `[[문서이름]]` 형태로 연결할 수 있어요:

```markdown
# 프로젝트 A 아키텍처

## 기술 스택
- 프론트엔드: [[React 컨벤션]]
- 백엔드: [[FastAPI 가이드]]
- 데이터베이스: [[PostgreSQL 설정]]

## 팀 구성
- 프론트엔드 담당: [[팀원 - 김철수]]
- 백엔드 담당: [[팀원 - 이영희]]
```

> **팁**: Obsidian을 사용하고 있다면 이 위키링크 형식이 동일해요! Obsidian 볼트를 통째로 가져올 수도 있어요. (자세한 내용은 5장 참고)

### 컬렉션으로 정리

위키 문서를 컬렉션(폴더)으로 묶어서 관리해요:

```
컬렉션 예시:
  📁 project-alpha/      ← 프로젝트 A 관련 문서
  📁 personal-notes/     ← 개인 메모
  📁 team-docs/          ← 팀 문서
  📁 tech-reference/     ← 기술 참조 자료
```

## 위키 검색하기 (search_wiki)

저장된 위키 문서를 찾는 방법이에요.

### 자연어 검색

```
"FastAPI 가이드 문서 찾아줘"
"PostgreSQL 설정 방법 위키에서 검색해줘"
"project-alpha 관련 문서 다 보여줘"
```

### 하이브리드 검색

`search_wiki`는 **키워드 검색 + 의미 검색(시맨틱 서치)**을 동시에 사용해요.

예를 들어 "데이터베이스 연결 문제"를 검색하면:
- "PostgreSQL connection error" 문서도 찾아줌 (의미가 같으니까)
- "DB 연결 설정" 문서도 찾아줌 (관련 개념이니까)

## 기억 삭제하기 (membase_forget)

잘못된 기억이나 더 이상 필요 없는 기억을 삭제해요.

```
"내가 Python을 싫어한다고 기억한 거 있으면 삭제해줘"
"프로젝트 B 관련 기억들 다 지워줘"
"이 위키 문서 삭제해줘"
```

> **주의**: 삭제 전에 AI가 확인 질문을 해요. 실수로 지우지 않도록!

## 자동 저장과 자동 불러오기

### 자동 저장 (Auto-Capture) 작동 방식

```
대화 중...
  나: "타입스크립트 좋아해"
  AI: "알겠어요!"
  나: "함수형 프로그래밍 스타일 써"  
  AI: "좋아요!"
  
  ← 5분 후 또는 20개 메시지 후 →
  
  자동으로 멤베이스에 저장:
  "사용자는 TypeScript와 함수형 프로그래밍 스타일을 선호함"
```

### 자동 불러오기 (Auto-Recall) 작동 방식

```
autoRecall: true 설정 시:

  나: "React 컴포넌트 만들어줘"
  
  [AI 내부 동작]
  search_memory("React 컴포넌트") 호출
  → 결과: "사용자는 함수형 컴포넌트, TypeScript, 훅 패턴 선호"
  → 이 정보를 컨텍스트에 자동 주입
  
  AI: "알겠어요! TypeScript 함수형 컴포넌트로 만들게요..." (개인화된 응답!)
```

### 스케줄러/크론 설정이 필요한가요?

**아니요!** 멤베이스는 완전 자동이에요.

- 자동 저장: 내부 타이머 (5분) 또는 메시지 카운터 (20개)로 작동
- 자동 불러오기: 각 AI 응답 생성 전에 자동 실행
- 외부 데이터(Gmail, Slack 등): 멤베이스가 알아서 주기적으로 동기화

추가 설정이나 관리가 필요 없어요.

## 대시보드에서 기억 관리하기

멤베이스 웹사이트([membase.so](https://membase.so))에서 로그인하면 대시보드를 볼 수 있어요.

### 대시보드에서 할 수 있는 것:

1. **기억 목록 보기**: 저장된 모든 메모리를 타임라인 형태로 확인
2. **지식 그래프 시각화**: 내 기억들 사이의 연결 관계를 그래프로 시각적으로 확인
3. **기억 수정/삭제**: 잘못된 기억을 직접 편집하거나 삭제
4. **위키 관리**: 위키 문서 목록 확인 및 편집
5. **연결된 앱 관리**: Gmail, Slack 등 연결된 외부 앱 확인

### 지식 그래프 시각화

대시보드에서는 내 기억들이 어떻게 연결되어 있는지 그래프로 볼 수 있어요. 예를 들어:

```
[나] ─── 사용 ──► [TypeScript]
[나] ─── 작업 ──► [프로젝트 A]
[프로젝트 A] ─── 포함 ──► [React]
[프로젝트 A] ─── 포함 ──► [FastAPI]
```

이런 시각화를 통해 내 기억들의 패턴을 한눈에 파악할 수 있어요.

## 실전 사용 시나리오

### 시나리오 1: 개발자 - 새 프로젝트 시작

```
나: "새 프로젝트 시작하려고 해. 내 스택 기억 있어?"

AI: (search_memory 호출)
   "네! 기억하고 있어요:
    - TypeScript + Python 선호
    - 프론트: React/Next.js
    - 백엔드: FastAPI  
    - DB: PostgreSQL
    - 배포: Vercel + Railway
    지난번 프로젝트에서 AWS Lambda 속도 문제가 있었던 것도 기억해요"
    
나: "이번엔 Supabase 써보려고"

AI: (add_memory 호출 - 자동)
   [내부에서 "사용자가 새 프로젝트에서 Supabase 시도 중" 저장]
```

### 시나리오 2: 블로거 - 글쓰기 보조

```
나: "새 블로그 포스트 써줘. 제목은 'AI 활용 팁'"

AI: (search_memory 호출)
   → "사용자의 글쓰기 스타일: 친근한 어조, 예시 많이 활용, 
      독자 수준: 초보자, 한국어 블로그, SEO 중요"
   
   AI가 이 스타일에 맞게 글 작성
```

### 시나리오 3: 창업자 - 투자자 미팅 준비

```
나: "내일 Series A 투자자 미팅인데 준비 도와줘"

AI: (search_memory 호출)
   → "회사: 멤베이스클론 / MAU: 5000명 / 목표: 1만 명 
      투자자: 황성준 (소프트뱅크) / 지난 미팅: 수익 모델 질문 많았음
      현재 MRR: 50만원 / 팀: 3명"
      
   AI가 맞춤형 투자자 미팅 준비 자료 제공
```

### 시나리오 4: 여러 AI 도구 간 전환

```
[Cursor에서 작업 중]
나: "Redis 캐싱 구현해줘. 저번에 했던 방식으로"
Cursor AI: (search_memory 호출)
   → "사용자는 이전에 Redis Cluster 대신 Sentinel 모드 사용, 
      TTL 기본값 3600초, Lua 스크립트 사용하지 않는 방식 선호"

[Claude Code로 전환]
나: "방금 Cursor에서 Redis 구현했어. 테스트 코드 만들어줘"
Claude: (search_memory 호출)
   → 같은 Redis 설정 정보 자동으로 인지!
   "Sentinel 모드 Redis에 맞는 pytest 테스트 코드 작성할게요"
```

## 핵심 요약

| 작업 | 방법 |
|------|------|
| 기억 저장 | AI에게 "기억해줘" 라고 말하거나 autoCapture 활성화 |
| 기억 검색 | AI에게 "기억나?" 라고 물어보거나 autoRecall 활성화 |
| 위키 저장 | AI에게 "위키에 저장해줘" 라고 요청 |
| 기억 삭제 | AI에게 "잊어줘" 라고 요청 |
| 대시보드 | membase.so 에 로그인해서 직접 관리 |
| 스케줄러 | 필요 없음! 완전 자동 |
