## 기본 사용 관련

### Q: 멤베이스에서 메모리와 위키의 차이가 뭔가요?

**A:**
- **메모리(Memory)**: "나"에 대한 개인 컨텍스트. 내 선호도, 과거 결정, 프로젝트 상태. 동적으로 변함.
  - 예: "TypeScript 선호", "이 프로젝트 PostgreSQL 써", "저번에 useEffect 버그 있었음"
  
- **위키(Wiki)**: 참조용 문서. 변경이 적은 공식 자료나 스펙.
  - 예: API 문서, 아키텍처 설계서, 팀 코딩 컨벤션

**기준**: "내 선호/결정/경험" → 메모리. "공식 문서/참조 자료" → 위키.

---

### Q: 멤베이스가 실수로 잘못된 정보를 저장했어요. 어떻게 고치나요?

**A:** 두 가지 방법이 있어요:

1. **AI에게 수정 요청**:
   ```
   "Python을 싫어한다고 저장된 기억이 있으면 삭제해줘. 
    사실은 Python을 좋아해"
   ```

2. **대시보드에서 직접 수정**:
   - [membase.so](https://membase.so) 로그인
   - Memory 탭에서 잘못된 기억 찾기
   - 수정(Edit) 또는 삭제(Delete) 클릭

---

### Q: 얼마나 많은 기억을 저장할 수 있나요?

**A:** 무료 플랜과 유료 플랜에 따라 저장 한도가 다를 수 있어요. 정확한 한도는 [membase.so](https://membase.so) 가격 페이지에서 확인하세요. 일반적으로 일상적인 사용에는 무료 플랜으로 충분해요.

---

### Q: 여러 AI 도구 (Claude + Cursor + ChatGPT)를 동시에 연결할 수 있나요?

**A:** 네! 이게 바로 멤베이스의 핵심 가치예요. 동일한 멤베이스 계정에 여러 AI 도구를 연결하면 모든 도구가 같은 메모리를 공유해요.

```
내 멤베이스 계정
  │
  ├── Claude Code (claude-membase 플러그인)
  ├── Cursor (cursor-membase 플러그인)
  ├── OpenClaw (openclaw-membase 플러그인)
  └── Hermes Agent (hermes-membase 플러그인)
  
모두 같은 메모리를 읽고 씀!
```

## 자동화 및 갱신 관련

### Q: 스케줄러나 크론(cron) 작업을 설정해야 하나요?

**A:** **아니요!** 멤베이스의 핵심 기능은 완전히 자동으로 작동해요.

| 기능 | 필요한 설정 |
|------|------------|
| 대화 자동 저장 | 없음 (autoCapture 기본 켜짐) |
| 기억 자동 불러오기 | 없음 또는 autoRecall 설정 한 번 |
| Gmail/Slack 동기화 | 최초 연결 한 번만 |
| 지식 그래프 업데이트 | 완전 자동 |

크론이 필요한 경우는 오직 **멤베이스 외부에 데이터를 백업**하고 싶을 때뿐이에요. 그것도 원한다면 선택적으로 설정하는 거예요.

---

### Q: autoCapture와 autoRecall의 차이가 뭔가요?

**A:**

**autoCapture (자동 저장):**
- 대화 내용을 자동으로 멤베이스에 저장
- 기본값: **켜짐 (true)**
- 저장 시점: 5분 비활성 후 또는 20개 메시지 축적 후
- 저장 안 되는 것: 50자 미만, `<private>` 태그 내용, 시크릿

**autoRecall (자동 불러오기):**
- AI 응답 전에 자동으로 관련 기억을 찾아서 컨텍스트에 주입
- 기본값: **꺼짐 (false)**
- 켜면: 모든 응답이 개인화됨, 컨텍스트 창 일부 사용
- 꺼두면: 필요할 때만 수동으로 `search_memory` 호출

---

### Q: 기억이 자동으로 업데이트되나요? 오래된 기억은 어떻게 되나요?

**A:** 멤베이스는 새 정보가 들어올 때 자동으로 기존 정보를 업데이트해요. 지식 그래프 구조 덕분에:

- 같은 주제에 대한 새 정보 → 기존 관계를 업데이트
- 충돌하는 정보 → 더 최근 정보를 우선 저장
- 오래된 정보 → 시간 스탬프로 구분, 최신 정보가 검색 우선순위 높음

예: "Python 좋아함"이 저장된 후 "이제 Go를 더 선호함"이 저장되면, 검색 결과에서 Go 선호가 더 최신 정보로 표시돼요.

---

### Q: 외부 앱(Gmail, Slack)은 얼마나 자주 동기화되나요?

**A:** 멤베이스가 내부적으로 주기적으로 동기화하며, 사용자가 따로 설정할 필요 없어요. 연동 앱에서 발생하는 중요한 이벤트(미팅 초대, 중요 이메일, Slack 결정 사항)가 감지되면 자동으로 메모리에 추가돼요.

## 저장 및 백업 관련

### Q: 데이터는 서버에만 저장되나요, 로컬에도 저장되나요?

**A:** 기본 멤베이스.so 서비스는 **클라우드 서버에만 저장**해요.

로컬에도 저장하고 싶다면:

1. **Hermes-Membase**: 로컬 에이전트 메모리 + 클라우드 동기화 (Mirror 기능)
2. **membase-cli export**: 수동 또는 스크립트로 로컬에 내보내기
3. **탈중앙화 버전 (membase-mcp)**: 블록체인에 저장, 로컬 제어 가능

---

### Q: 멤베이스 서비스가 종료되거나 계정을 삭제하면 데이터를 잃어버리나요?

**A:** 멤베이스의 정책은 "당신의 데이터는 항상 당신 것"이에요. 다음 방법으로 데이터를 내보낼 수 있어요:

```bash
# 모든 데이터 내보내기
membase export --output all-my-data.json

# 위키만 마크다운으로 내보내기
membase export --type wiki --format markdown --output my-wiki/
```

**서비스 중단 전 대비책:**
1. 정기적으로 `membase export` 실행
2. GitHub Actions로 자동 백업 설정 (4장 참고)
3. 탈중앙화 버전 병행 사용 고려

---

### Q: GitHub에 동시에 백업할 수 있나요?

**A:** 직접적인 GitHub 동기화 기능은 내장되어 있지 않지만, 다음 방법으로 구현할 수 있어요:

**방법 1: GitHub Actions (권장)**
```yaml
# .github/workflows/backup.yml
on:
  schedule:
    - cron: '0 0 * * 0'  # 매주 일요일
jobs:
  backup:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: membase export --output backup.json
      - run: git add backup.json && git commit -m "Weekly backup" && git push
```

**방법 2: 로컬 스크립트 + Git**
```bash
membase export --output ~/membase-backups/backup-$(date +%Y%m%d).json
cd ~/my-backup-repo
git add . && git commit -m "Backup" && git push
```

## 외부 도구 연동 관련

### Q: 옵시디언(Obsidian)과 어떻게 연동하나요?

**A:** 두 가지 방향으로 연동 가능해요:

**방향 1: 옵시디언 → 멤베이스 (가져오기)**
- 기존 옵시디언 볼트를 멤베이스 위키로 가져옴
- 대시보드의 "Import from Obsidian" 기능 사용
- 위키링크(`[[문서명]]`) 형식이 그대로 유지됨

**방향 2: 멤베이스 → 옵시디언 (내보내기)**
- 멤베이스 위키를 마크다운으로 내보내기
- 내보낸 파일을 옵시디언 볼트에 넣기
- Obsidian Git 플러그인으로 버전 관리

**실용 팁**: 멤베이스 위키 형식이 옵시디언 형식과 거의 동일해요 (마크다운 + `[[위키링크]]`). 두 도구 사이에 데이터 이동이 쉬워요.

---

### Q: ChatGPT와도 연동 가능한가요?

**A:** ChatGPT는 현재 MCP를 직접 지원하지 않아요. 하지만 다음 방법으로 활용할 수 있어요:

1. **ChatGPT Custom GPT + API**: 멤베이스 API를 통해 Custom GPT의 액션으로 설정
2. **수동 공유**: ChatGPT와 대화하기 전 `membase://recent`나 `search_memory` 결과를 복사해서 붙여넣기
3. **ChatGPT 대화 가져오기**: 이전 ChatGPT 대화를 멤베이스로 가져올 수 있음

---

### Q: Notion과 연동할 수 있나요?

**A:** 현재 Notion은 멤베이스의 외부 앱 연동 목록에 언급되지 않아요. 하지만:
- Notion 페이지를 마크다운으로 내보내서 멤베이스 위키에 수동으로 가져올 수 있어요
- 멤베이스의 지속적인 서비스 개선에 따라 Notion 통합이 추가될 수 있어요

공식 사이트([membase.so](https://membase.so))에서 최신 연동 현황을 확인하세요.

## 보안 및 프라이버시 관련

### Q: AI 에이전트가 실수로 비밀번호를 저장하면 어떻게 되나요?

**A:** 멤베이스는 몇 가지 자동 보호 기능을 제공해요:

1. **자동 필터링**: 일반적인 API 키 패턴, 환경 변수 형식은 자동으로 제외
2. **`<private>` 태그**: 태그로 감싼 내용은 절대 저장 안 됨
3. **수동 삭제**: 실수로 저장됐다면 `membase_forget`으로 즉시 삭제

**예방 방법**: CLAUDE.md나 .cursorrules에 "API 키, 비밀번호, 시크릿은 절대 멤베이스에 저장하지 마세요"를 명시해두세요.

---

### Q: 내 기억 데이터가 AI 모델 학습에 사용되나요?

**A:** 멤베이스의 서비스 약관(Terms of Service)에 따르면, 데이터 소유권은 사용자에게 있어요. 정확한 데이터 사용 정책은 [membase.so/terms](https://membase.so/terms)에서 확인하세요.

---

### Q: 팀과 기억을 공유할 수 있나요?

**A:** 멤베이스는 기본적으로 개인 메모리 시스템이에요. 팀 공유 기능에 대해서는 공식 사이트에서 최신 정보를 확인하세요. 현재 멀티 에이전트 조율(Multi-Agent Coordination) 기능이 제공되어 여러 AI 에이전트가 같은 계정의 메모리를 공유할 수 있어요.

## 문제 해결 관련

### Q: AI 에이전트가 기억을 잘 활용하지 않아요

**A:** 다음을 확인해보세요:

1. **autoRecall 설정 확인**: `autoRecall: true` 로 설정했는지 확인
2. **CLAUDE.md/cursorrules 확인**: 에이전트에게 명시적으로 search_memory를 호출하도록 지시했는지 확인
3. **기억이 저장되어 있는지 확인**: 대시보드에서 실제로 메모리가 있는지 확인
4. **검색어 다양화**: `search_memory("코딩 선호도")`보다 `search_memory("Python TypeScript 선호")`처럼 더 구체적으로

---

### Q: "Connection failed" 오류가 계속 나와요

**A:**
1. 인터넷 연결 상태 확인
2. OAuth 토큰이 만료됐을 수 있음 → 재로그인: `openclaw membase login` 또는 `/membase:login`
3. 멤베이스 서비스 상태 확인: [membase.so](https://membase.so) 접속

---

### Q: 기억이 저장은 됐는데 검색이 안 돼요

**A:**
1. 검색어를 바꿔보세요 (동의어, 다른 표현 사용)
2. 대시보드에서 직접 메모리 목록 확인
3. 저장 직후에는 인덱싱 시간이 필요할 수 있어요 (보통 수 초 이내)
